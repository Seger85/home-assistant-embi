# 13 – Documentation and Community Contract

## 1. README goal

A new user should understand EMBi in under three minutes:

1. what EMBi is,
2. why it exists,
3. what it can do,
4. a simple screenshot or diagram,
5. installation,
6. first setup.

## 2. Product statement

German meaning:

> EMBi bindet Emby-Clients als Media-Player in Home Assistant ein und hilft optional dabei, alte Geräte-Einträge und nicht gewünschte Home-Assistant-Player sicher zu bereinigen.

English meaning:

> EMBi adds Emby clients to Home Assistant as media players and can optionally clean up old device records and unwanted Home Assistant players safely.

## 3. Required examples

- family users grouped under names,
- a shared television with all known users,
- technical access such as Home Assistant or Homarr,
- removing a player from Home Assistant while preserving Emby history,
- server-history cleanup as a separate action.

## 4. Required sections

- Features
- Installation with HACS
- Initial configuration
- Devices & players
- Users and shared devices
- Technical access
- Cleanup
- Safety rules
- Restore a hidden player
- Troubleshooting
- Diagnostics
- Privacy
- Development
- Changelog link

## 5. Version history placement

Do not make README primarily a release journal. Version-specific details belong in GitHub Releases, CHANGELOG and migration notes.

## 6. Forum readiness

Provide a short forum-ready introduction without implementation jargon, using screenshots, explaining historical clients, user grouping, HACS installation and limitations honestly.
